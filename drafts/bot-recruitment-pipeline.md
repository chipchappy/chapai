# ChapAI Bot Recruitment Pipeline

Goal: add only the next specialist employee lanes that increase profit, shipping speed, or content throughput after the current product is stable.

## Recruitment order

### 1. Growth Studio Bot
- Owns: creator outreach, short-form posts, founder DMs, affiliate asks, launch copy, and basic funnel experiments.
- Why next: fastest path to more clicks, more trials, and the first recurring revenue.
- Minimum brain requirements:
  - one durable `target creator` list
  - one reusable `DM pack`
  - one reusable `post pack`
  - one `offer menu`
  - one `weekly learnings` note
- Persistence rule:
  - keep only approved outreach patterns, best hooks, and reply summaries
  - discard raw noise after it is summarized

### 2. Content Ops Bot
- Owns: question batch promotion, schema validation, exam mix balancing, and content freshness.
- Why next: keeps CCRN and NCLEX growing without expensive manual review.
- Minimum brain requirements:
  - one `question schema` memory
  - one `category map` for CCRN and NCLEX
  - one `batch status` note
  - one `promotion rules` note
- Persistence rule:
  - store only durable generation patterns and validation failures that changed future output

### 3. Conversion QA Bot
- Owns: homepage, package pages, checkout flow, success flow, and drop-off fixes.
- Why next: small UI changes often beat big feature work for profit.
- Minimum brain requirements:
  - one `conversion checklist`
  - one `known blockers` note
  - one `best-performing CTA` note
  - one `before/after` log
- Persistence rule:
  - keep only issues that affected clicks, checkout, or trust

### 4. Support / Onboarding Bot
- Owns: first-buy welcome flow, account help, billing questions, and retention nudges.
- Why next: keeps new buyers from churning and reduces manual support load.
- Minimum brain requirements:
  - one `support playbook`
  - one `common questions` note
  - one `escalation rule` note
  - one `post-purchase checklist`
- Persistence rule:
  - keep the shortest useful answer for each repeated support case

### 5. Product Scout Bot
- Owns: next product ideas, adjacent nurse tools, and which market is worth building after ChapAI is stable.
- Why next: helps us pick the next revenue lane without wandering.
- Minimum brain requirements:
  - one `scout shortlist`
  - one `market scorecard`
  - one `build-vs-buy` note
  - one `profit potential` ranking
- Persistence rule:
  - keep only validated opportunities with a clear buyer and low launch complexity

### 6. Automation / Swarm Bot
- Owns: repeated agent workflows, queue hygiene, persistence checks, and low-token task routing.
- Why next: keeps the whole system cheaper, steadier, and easier to scale.
- Minimum brain requirements:
  - one `workflow contract`
  - one `task routing` note
  - one `failure mode` note
  - one `restart continuity` note
- Persistence rule:
  - keep process rules, not long reasoning traces

## Brain standard for every new bot
- one `identity` record
- one `role` record
- one `skills` list
- one `goals` list
- one `durable memory` list
- one `recent events` list
- one `current task` record
- one `blockers` record
- one `last updated` timestamp

## Promotion rule
- A bot only gets added if it can:
  - own a real money or speed lever
  - keep a small persistent brain
  - work from a bounded queue
  - improve the next cycle without needing a huge context dump

## Best immediate hire after current stability
- `Growth Studio Bot`
- It has the fastest path to sales, content distribution, and first recurring profit.
