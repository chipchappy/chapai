## Hume Content Review Playbook

### Mission
Turn cheap batch growth into premium exam prep by reviewing promoted batches for:
- rationale quality
- diagram opportunities
- taxonomy balance
- bedside realism
- exam-style priority framing

### Review order
1. Inspect the latest promoted mixed batch.
2. Check for weak or generic rationales.
3. Tag any question that needs a diagram or visual rationale.
4. Flag duplicate-feeling stems or weak distractors.
5. Compare category mix against current draft/live totals.

### Elite question rules
- The right answer should feel inevitable after the learner sees the key clue.
- Distractors should fail for a specific clinical reason, not because they are vague.
- Rationales should name the physiologic priority in plain language.
- NCLEX items should bias toward safety, delegation, pharm, prioritization, and what changes the next action.
- CCRN items should bias toward hemodynamics, oxygenation, perfusion, trend recognition, and escalation timing.

### Diagram triggers
Add or improve a visual rationale when the learner benefits from:
- trend recognition
- pressure or flow reasoning
- ventilation or oxygenation pathways
- perfusion or shock comparison
- prioritization flow

### Output standard
Every review pass should leave behind:
- one concise summary of the latest batch quality
- one note about rationale drift or improvement
- one note about new diagram candidates
- one note about category or difficulty balance

### QA sampler v1
- Sample 20% of every promoted batch, minimum `8` items, maximum `24` items.
- For the first `50` live candidates in any NCLEX priority category, review `100%` before live promotion.
- Score every sampled question on five axes:
  - `rationale depth`: pass only if the rationale is at least 3 sentences and names the physiologic or safety mechanism.
  - `distractor specificity`: pass only if the distractors fail for named clinical reasons, not generic wording.
  - `topic gap contribution`: flag the item if it belongs to a category with thin live coverage or weak recent growth.
  - `diagram worthiness`: tag when trends, vent settings, shock comparison, perfusion, oxygenation, or prioritization flow would teach faster visually.
  - `rewrite priority`: `0 = pass`, `1 = minor polish`, `2 = rationale rewrite`, `3 = stem plus distractor rebuild`.
- Any sampled item scoring `3` is a hard rewrite candidate.
- If more than `30%` of the sampled items score `2` or `3`, reject the batch back into rewrite routing instead of silently compounding it.
- Use rejection reasons:
  - `rationale_drift`
  - `distractor_weakness`
  - `stem_vagueness`

### Daily review artifacts
Every Hume review pass should leave behind:
- one machine-readable latest review file
- one concise markdown report
- one live-promotion candidate slate
- one memory event describing the strongest drift or improvement signal
