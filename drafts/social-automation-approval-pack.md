# Mercury Social Automation Approval Pack

## Goal
Move Mercury from `manual-ready` to `semi-auto`, then to `api-ready`, without risking spam or sloppy posting.

## What Mercury can safely do now
- maintain reply banks
- maintain creator shortlists
- package founder posts for X and Instagram
- update trend watchlists
- suggest the next highest-leverage distribution wave

## What Mercury needs before semi-auto
- one approved X posting path
- one approved Instagram publishing path
- posting cadence limits
- one reviewed content queue
- one "never post these" ruleset

## Safe cadence proposal
- X founder posts: max 1 per day
- X replies: max 6 per day
- Instagram founder feed: max 3 per week
- Instagram story ideas: package only until publishing is trusted

## Review rules
- no fake testimonials
- no spammy reply chains
- no more than one CTA in a short reply
- prefer useful, human, warm language
- if a post reads too "AI," do not publish

## Approval checklist
- [ ] X write credentials available
- [ ] Instagram professional publishing credentials available
- [ ] posting cadence approved
- [ ] blocked phrase list approved
- [ ] reviewed queue path approved

## First semi-auto pilot
1. Mercury drafts three X founder posts.
2. Mercury drafts five warm replies.
3. Human approves the queue.
4. Mercury publishes only the approved queue.
5. Mercury logs reply/click signals and updates the trend watchlist.
